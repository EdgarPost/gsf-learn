グリーンソフトウェア ムーブメントに参加してみませんか？

ソフトウェアアプリケーションをより持続可能なものにしたい、グリーンソフトウェアでキャリアを築きたいとお考えの方には、このトレーニングがおすすめです。このトレーニングでは、アプリケーションのドメイン、業界や組織の規模や種類、プログラミング言語、フレームワークに関係なく、より環境に優しいアプリケーションを構築、維持、運用する方法を学ぶことができます。

[関連する試験](https://grnsft.org/practitioner/lf-exam)に合格すると、Linux Foundationから修了証が発行されます。あなたの勤務先によっては、本コースを修了するための単位取得を容易にするためのフォームを作成しているところもあります。あなたの勤務先がこのリストに含まれている場合は、あなたの組織名をクリックしてあなたの情報を登録してください。あなたの組織が自組織のフォームをリストに追加したいと考えている場合は、help@greensoftware.foundationで当チームにご連絡ください。

## ソフトウェア プラクティショナー（実践者）とは？

ソフトウェア プラクティショナーとは、ソフトウェアアプリケーションの構築、展開および管理の実務に携わる人のことです。ソフトウェア・プラクティショナーとしては以下が挙げられますが、これらに限定されるものではありません。

- 開発者
- DevOps（開発・運用）
- テスター（試験者）
- 設計者
- SRE
- プロダクトマネージャー
- UX設計者

## このコースを最大限に活用するには？

- ソフトウェアアプリケーションの構築、展開または管理の業界経験が1年以上あること。
- 各セクションを最初から最後まで順番に完了すること。
- 各セクションの最後にある知識チェックを完了すること。
- 修了証の取得を希望する場合は、Linux Foundationを通じて試験を受けること。

## このトレーニングを修了するころには、次のことが理解できるようになります

- ソフトウェア・アプリケーションにおける炭素排出の主な要因。
- ご使用中のソフトウェアの二酸化炭素排出量を削減するためにできる3つのアクション。
- 温室効果ガス（GHG）プロトコルまたはソフトウェア炭素強度（SCI）仕様を用いてソフトウェアの排出量を測定する方法。
- さまざまなタイプの企業環境目標があるため、グリーンソフトウェアの提案を調整して、組織の公約を達成することができます。
